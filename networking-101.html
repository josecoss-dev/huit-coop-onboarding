01
What Is a Network, Actually?
Foundation
The Big Picture

A network is any group of devices that can communicate. Your laptop, a printer down the hall, Harvard's servers — all nodes. The internet is just a massive network of networks tied together by agreed-upon rules called protocols.

🏙️ City Analogy

Buildings = devices. Streets = cables and Wi-Fi. Addresses = IP addresses. The post office = DNS. Traffic rules = protocols. Simple as that.

The 6 Concepts You Must Know
📍
IP Address
Logical address used to route data across a network. Assigned dynamically — can change.
10.0.2.55
🪪
MAC Address
Physical ID burned into the network card at the factory. Permanent. Unique to that hardware.
A4:83:E7:2C:11:F0
📖
DNS
Translates domain names to IP addresses. You type google.com, DNS finds the number.
8.8.8.8
🏷️
DHCP
Server that hands out IP addresses automatically when a device connects. Nothing to configure.
Auto-assign
🚪
Gateway
The router connecting your local network to the rest of the internet. All outbound traffic flows through it.
10.0.0.1 (typical)
🔒
Subnet
A segmented slice of a larger network. Harvard separates departments into their own subnets.
/24 = 254 hosts

MAC addresses only work on your local segment. IP addresses work everywhere. ClearPass identifies your hardware by MAC — that's why every device needs its own registration.

✓ Key Takeaway

IP = logical, can change. MAC = physical, permanent. ClearPass authenticates devices by MAC address, not IP.

What is Aruba ClearPass?

ClearPass is Harvard's Network Access Control (NAC) system. Every device trying to connect to Harvard Secure passes through it. ClearPass checks: Is this device registered? Is this user authorized? It then assigns the right VLAN for your role and department.

03
Finding Your MAC Address
Hands-On

You need the device's Wi-Fi MAC address before you can register it. Here's how to find it on everything you'll touch at Harvard.

Method A — System Settings
  1. Click the Apple menu → System Settings
  2. Click Wi-Fi in the sidebar
  3. Click Details… next to your connected network
  4. Look for Wi-Fi Address — format: A4:83:E7:2C:11:F0
Method B — Terminal (Faster)
  1. Spotlight → type terminal → open it
  2. Run: ifconfig en0 | grep ether
  3. The ether line is your Wi-Fi MAC address
💡 Pro Tip

en0 is almost always Wi-Fi on Macs. Ethernet adapters are usually en1 or higher. When in doubt, confirm in System Settings first.

Method A — Settings
  1. Press Win + I to open Settings
  2. Go to Network & Internet → Wi-Fi
  3. Click your connected network name
  4. Scroll to find Physical address (MAC)
Method B — Command Prompt (Faster)
  1. Press Win + R → type cmd → Enter
  2. Run: ipconfig /all
  3. Find Wireless LAN adapter Wi-Fi section
  4. Look for Physical Address — that's your MAC
💡 Dell Shortcut

Many Dell laptops have a sticker on the bottom labeled "WLAN MAC" — useful when the device won't boot or is locked out of the network entirely.

  1. Go to Settings → General → About
  2. Scroll to find Wi-Fi Address
  3. That's your MAC — copy it before going to the portal
⚠ Critical — Private Wi-Fi Address

iOS rotates the MAC address per network by default. This BREAKS ClearPass. Before registering an iPhone: Settings → Wi-Fi → ⓘ next to Harvard Secure → toggle Private Wi-Fi Address OFF. Without this step, the device will keep losing access.

  1. Connect to Harvard University — the open guest network, no password. This gives you internet to reach the portal.
  2. Open a browser and go to getonline.harvard.edu
  3. Sign in with your HarvardKey credentials
  4. Download the certificate when prompted — this cert is what authenticates the device on Harvard Secure
  5. You'll be asked to sign in a second time — this is expected and part of the cert installation flow
  6. Install the certificate:
    Mac: Opens Keychain Access → click Add → then right-click cert → Get Info → Trust → Always Trust Dell/Windows: Double-click the file → Certificate Import Wizard → follow prompts
  7. Disconnect from Harvard University, connect to Harvard Secure — authenticates automatically via cert
⚠ Mac — Trust the Certificate

Even after install, macOS won't trust it by default. Open Keychain Access → find the Harvard cert → double-click → Trust → set to Always Trust. Without this step, Wi-Fi still fails.

✓ Pre-imaged Dell?

If the machine already has the certificate (HUIT-imaged devices), skip to step 7. Just connect to Harvard Secure — cert handles the rest.

⚠ Use Ethernet MAC — Not Wi-Fi MAC

Every device has separate MACs for Wi-Fi and Ethernet. For wired registration, you need the Ethernet MAC. Mac: System Settings → Network → Ethernet → Details → Hardware. Windows: ipconfig /all → look under "Ethernet adapter" section.

ClearPass Admin Portal
  1. Log into ClearPass admin portal with your HUIT tech credentials
  2. Navigate to device registration → Create Device
  3. Enter the MAC address manually: format 01:23:45:AB:CD:EF
  4. Set Owner to user's HarvardKey email or HUID
  5. Set Role under Mobility — controls VLAN assignment. Most users: Mobility Admin
  6. Set MAC Filter — Registered for most locations, HCL for Library
  7. Set Expiration — co-op end date, or 1 year for permanent staff
  8. Save. Device active in 1–2 minutes.
⚠ Access Note

You may not have admin portal access on day 1. If locked out, escalate to your supervisor. Register your own device via getonline.harvard.edu first.

Common Errors
🚫
"Device already registered"
MAC already in the system — prior semester or old employee. Admin can reassign or delete the old entry.
🔄
"Invalid MAC address"
Mistyped, or iOS Private Address is rotating. Verify format — colons or dashes both work.
⏳
Registered but still blocked
Propagation delay. Forget network, wait 2 min, reconnect. Still failing after 5 min → escalate.
🔑
HarvardKey error on portal
Account not provisioned or password mismatch. Verify HarvardKey activation email was completed.

Real-World Troubleshooting Scenarios
Applied

Four scenarios straight from your three areas. Read each one, think through the answer, then reveal it.

📞
Scenario 1 — The New Hire
Lamont Library

"A new staff member just got their laptop today. They can see Harvard Secure in the Wi-Fi list but authentication keeps failing."

Your move: Device has no certificate. Connect to Harvard University, go to getonline.harvard.edu, complete the certificate flow, then reconnect to Harvard Secure. Confirm HarvardKey is activated first.

📞
Scenario 2 — Works on Phone, Not Laptop
Art Museum

"A curator's iPhone works fine on Harvard Secure. Their new MacBook won't connect at all."

Your move: Different device = different MAC. The phone has a certificate; the Mac doesn't. Connect Mac to Harvard University, run through getonline.harvard.edu, install the certificate. Each device needs its own cert.

📞
Scenario 3 — Was Working, Now Isn't
Divinity School

"A professor's laptop worked fine for a year. They went on sabbatical, came back, and now can't connect. Nothing changed."

Your move: Device registration expired — most run 1 year. Check ClearPass for the device and renew. Extremely common after summer or extended leave. Also verify the cert hasn't expired.

📞
Scenario 4 — Random iPhone Disconnects
Any Location

"A student registered their iPhone last week. Now it randomly drops Harvard Secure every day."

Your move: iOS Private Wi-Fi Address is rotating the MAC. ClearPass sees a different address and blocks it. Fix: Settings → Wi-Fi → ⓘ → Private Wi-Fi Address → OFF for Harvard Secure. Then re-register with the real MAC.
