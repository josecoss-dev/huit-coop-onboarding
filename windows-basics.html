User Accounts, UAC & LAPS — The Admin Flow
Critical
Three Account Types You'll Deal With
👤
Standard User
The default for most Harvard staff. Can use software but can't install, change system settings, or approve UAC prompts. Most faculty and staff are here.
🔑
Local Admin User
Some power users and co-ops have local admin rights on their machine. They can approve their own UAC prompts. UAC still pops but they can click Yes.
🛡️
HUITadmin Account
Shared local admin account on every Harvard machine. Password is unique per machine and rotates — managed by LAPS. Retrieved from HUITtoolkit.exe.
What Is UAC?

User Account Control (UAC) is Windows' permission gate. Whenever something tries to make a system-level change — install software, modify settings, run certain commands — Windows throws a prompt. Standard users see a credential prompt (enter admin username + password). Admins see a consent prompt (just Yes/No). You will see this constantly.

The LAPS Flow — When UAC Blocks a Standard User

This is the workflow you'll run dozens of times. A user gets a UAC prompt they can't get past. Here's exactly what to do:

① UAC Appears
User is blocked
Windows shows a credential prompt. The user's standard account can't proceed. They call you.
→
② Open HUITtoolkit
Run HUITtoolkit.exe
Launch the HUIT toolkit. Navigate to the LAPS section. Search for the machine by hostname or asset tag.
→
③ Pull LAPS Password
Copy the password
The toolkit shows the current LAPS password for that machine. This is a randomly generated, rotating password unique to this machine.
→
④ Enter Credentials
Username: HUITadmin
In the UAC prompt, type HUITadmin as the username and paste the LAPS password. Action proceeds.
⚠ Critical — LAPS Passwords Are Temporary

LAPS passwords rotate on a schedule. Don't write them down or share them after use. Pull a fresh one each time. The password displayed in HUITtoolkit is only valid until the next rotation — could be hours or days. Always pull fresh.

Show Hidden Files & Extensions

By default Windows hides file extensions and hidden folders. You need to turn these on — you'll miss problems and can't see critical system files without them.

  1. Open File Explorer → Click the View menu (or three-dot menu in Win11)
  2. Select Show → check Hidden items
  3. In the same menu, check File name extensions
  4. Now you can see AppData, .dll files, .bat files, etc.
💡 Quick Way

In any Explorer window: Alt + V then H toggles hidden files. Or use the View ribbon. Keep this on permanently on your work machine — there's almost never a reason to hide extensions as a tech.

Reading Each Tab

Processes shows everything running. Sort by CPU, Memory, or Disk (click column headers) to find the culprit immediately. Columns in red = that resource is critically high.

🔴
High CPU
SearchIndexer.exe, antivirus scans, Windows Update, and runaway browser tabs are common culprits. If >90% sustained, investigate.
🟡
High Memory
Teams, Chrome, and Outlook each use 500MB–2GB. If RAM is maxed, machine will use disk as swap = extreme slowness.
🔵
High Disk
100% disk usage = a very common Win11 issue. Often SearchIndexer.exe, Windows Update, or Superfetch/SysMain. Check via Processes tab.
⚠ Don't Kill These

csrss.exe, lsass.exe, winlogon.exe, svchost.exe, and System are critical system processes. Ending them crashes Windows. If they're high CPU, something else is wrong — don't kill them directly.

Performance gives you real-time graphs and hardware info. The most useful: CPU speed (shows if it's throttling), RAM in use, Disk I/O, and at the bottom — how long the machine has been on since last reboot. "Open Resource Monitor" at the bottom gives even more detail.

✓ First Diagnostic Question

Always check uptime. If the machine hasn't been rebooted in 15+ days, reboot it first. Many "slow computer" and "weird behavior" issues resolve immediately after a restart. Most users never turn their machines off.

Startup lists every app that launches when Windows starts. This is a goldmine for fixing slow boot times. Each item shows its "Startup impact" — High, Medium, or Low. Disable the High-impact ones that aren't needed (right-click → Disable).

💡 What to Disable

Safe to disable: Spotify, Discord, Slack (if not required), OneDrive (if causing issues), Teams (it auto-starts when opened). Don't disable: CrowdStrike, security tools, Dell management agents, anything from HUIT.

Services shows every Windows service. Right-click → Start/Stop/Restart. This is where you restart Windows Search, Windows Update, Print Spooler, etc. without rebooting. Critical for fixing stuck services without kicking the user off.

Services to know: WSearch (Windows Search/Index), wuauserv (Windows Update), Spooler (Print), CryptSvc (Certificates), BITS (Background transfers).

Windows Search & Indexing — Why It Fails & How to Fix It
Common Issue
How Search Indexing Works

Windows maintains a database of all your files, file contents, and metadata so that when you search, results are instant rather than scanning the whole disk. SearchIndexer.exe is the background process that builds and maintains this database continuously.

📁
Your Files
C:\Users\...\Documents
→
⚙️
SearchIndexer.exe
Reads & catalogs files
→
🗄️
Search Index DB
C:\ProgramData\Microsoft\Search
When SearchIndexer.exe Eats Your CPU/Disk

This is extremely common. A machine running at 80-100% disk usage that's "brand new" is almost always the search indexer building the initial index. This is normal the first time but if it's been running for days, something is wrong.

🆕
New Machine / New Profile
Initial indexing takes 1–4 hours. Normal — let it run. CPU and disk will be high. Machine is usable, just slower.
💀
Corrupted Index
Search returns wrong results or nothing. Indexer loops forever. Fix: rebuild the index. This takes 1–2 hours.
📡
Network Drive Indexed
If a mapped network drive or cloud sync folder is in the indexed locations, SearchIndexer hammers the network. Remove network paths from indexed locations.
🔄
Service Crash Loop
WSearch service keeps crashing and restarting. Check Event Viewer Application log for SearchIndexer errors. Usually fixed by rebuilding.
Rebuild the Search Index
  1. Win + R → type control → Enter → find Indexing Options (or search for it in Control Panel)
  2. Click Advanced button (requires admin — use HUITadmin + LAPS if needed)
  3. Click Rebuild — confirm the warning. This deletes and recreates the entire index
  4. Click OK. Indexing starts fresh — takes 1–4 hours depending on number of files
  5. Machine can be used during this time but search will be incomplete until done
⚡ Faster Alternative — PowerShell

Stop the service, delete the database, restart it:
Stop-Service WSearch -Force
Remove-Item -Path "C:\ProgramData\Microsoft\Search\Data\Applications\Windows" -Recurse -Force
Start-Service WSearch
Run as admin. Rebuild starts automatically.

Check & Fix Indexed Locations
  1. Control Panel → Indexing Options → Modify
  2. Review what's listed. You should see: C:\Users\[name], Offline Files, Start Menu
  3. If you see mapped drives (\\server\share), OneDrive folders with many files, or other network paths — uncheck them
  4. Click OK. Index rebuilds from only the correct locations

Common Windows Fixes — Your Troubleshooting Playbook
Critical

Click each scenario to expand the full fix procedure. These are the calls you'll get in your first week.

🐌
Computer Is Very Slow — User Can Barely Work
High Impact
  1. Reboot first. Ask when it was last restarted — if >7 days, reboot immediately. This alone fixes 30% of "slow computer" calls.
  2. Open Task Manager (Ctrl+Shift+Esc) → sort by CPU, then Memory, then Disk. Find the top consumer.
  3. If SearchIndexer.exe is top: it's indexing. If it's been running hours, rebuild the index (Module 5).
  4. If Disk is at 100%: check if Windows Update is running. Also check SysMain service — disable it if it's hammering disk on an SSD.
  5. Check Startup tab — disable high-impact non-essential apps (Spotify, Teams if not required, etc.).
  6. Check available disk space: Get-Volume. Under 10% free causes severe slowness. Clean temp files, empty Recycle Bin, run Storage Sense.
  7. If RAM is maxed: check if user has 50 browser tabs open. Also check if the machine only has 8GB RAM — may need upgrade request via ticket.
Disable-Service SysMain # If SysMain is thrashing disk on SSD
Remove-Item $env:TEMP\* -Recurse -Force -EA SilentlyContinue # Clear temp
🔄
Windows Update Stuck or Failing
High Impact
  1. Check Settings → Windows Update → View update history. Note the error code if shown (e.g. 0x80070422, 0x800f0905).
  2. Stop the Windows Update services, clear the download cache, restart:
    Stop-Service wuauserv, bits, cryptsvc -Force
    Rename-Item C:\Windows\SoftwareDistribution SoftwareDistribution.old
    Start-Service wuauserv, bits, cryptsvc
  3. Try updating again from Settings → Windows Update → Check for updates.
  4. If still failing: Run DISM then SFC:
    DISM /Online /Cleanup-Image /RestoreHealth
    sfc /scannow
  5. Check that the machine has at least 10GB free disk (updates need staging space).
  6. If machine is on SCCM: updates may be deployed through Software Center — check there instead of Windows Update directly.
💡 SCCM Machines

On Harvard SCCM-managed machines, updates often come through Software Center, not direct Windows Update. Check Software Center → Updates tab first before manually touching Windows Update.

💥
App Won't Open / Crashes Immediately
Medium
  1. Check if the app is already running in Task Manager (processes tab) but just not showing. End task on any existing instance, then relaunch.
  2. Check Event Viewer → Windows Logs → Application for errors with the app name at the crash time. Note the Event ID and exception code.
  3. Clear the app's temp/cache folder: navigate to %LOCALAPPDATA%\[AppName]\ and delete the cache subfolder.
  4. For Office apps: try running in safe mode first — winword /safe, excel /safe. If it works, a COM add-in is the problem. Disable add-ins in Options.
  5. For Teams: fully quit Teams from system tray → delete %LOCALAPPDATA%\Packages\MSTeams* → restart Teams.
  6. If none of the above: repair or reinstall via SCCM Software Center. Don't manually download — use approved sources only.
💾
Running Out of Disk Space
Medium
  1. Run Storage Sense: Settings → System → Storage → "Run Storage Sense now". Cleans temp files, Downloads older than 30 days, Recycle Bin.
  2. Manually clear temp: Remove-Item $env:TEMP\* -Recurse -Force -EA SilentlyContinue and Remove-Item C:\Windows\Temp\* -Recurse -Force -EA SilentlyContinue
  3. Check for Windows.old folder at C:\ — a previous Windows install that wasn't cleaned up. Can be 20–40GB. Remove via Disk Cleanup → Clean up system files → Previous Windows installations.
  4. Check C:\Windows\Installer — sometimes bloats to 10+ GB. Don't delete manually. Use msizap or DISM to clean.
  5. Look for massive user files: run Get-ChildItem C:\Users -Recurse -File | Sort Length -Desc | Select -First 20 | Select Name, Length, FullName
  6. If OneDrive shows many local files, check if Files On Demand is configured — files should stay in the cloud until needed.
👤
User Can't Log In / Profile Corruption
High Impact
  1. Check if they can log into another machine with the same credentials. If yes, the issue is local profile not domain account.
  2. Log in as HUITadmin using LAPS password. Check C:\Users\ — look for joc5799.DOMAIN or joc5799.000 folders — these are corrupted temporary profiles.
  3. In Registry: HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList — find the SID entry that points to the corrupt profile. Note the path.
  4. Backup user data: copy Documents, Desktop, Downloads, AppData\Roaming from the old profile to a safe location or OneDrive.
  5. Delete the corrupt profile entry in Registry and in C:\Users\. User logs in fresh, gets a new profile.
  6. Restore backed-up data to the new profile location.
⚠ Always Back Up First

Never delete a user profile without backing up their data first. Check if they use OneDrive — if Desktop and Documents are synced to OneDrive, recovery is much simpler.

🖨️
Printer Not Working / Print Queue Stuck
Quick Fix
  1. Clear the print queue: Stop-Service Spooler -Force → Remove-Item C:\Windows\System32\spool\PRINTERS\* -Force → Start-Service Spooler
  2. If printer is offline: check it's powered on and connected. Try removing and re-adding via Settings → Bluetooth & devices → Printers.
  3. For network printers: ping the printer IP. If no response, the printer or its network connection is down.
  4. Driver issue: Device Manager → find the printer → right-click → Update driver. Or remove and reinstall via Add Printer wizard.
Stop-Service Spooler -Force
Remove-Item C:\Windows\System32\spool\PRINTERS\* -Force -EA SilentlyContinue
Start-Service Spooler
# Queue is now cleared

Dell Command Update — Keeping Machines Patched
Hardware
What Dell Command Update Does

Dell Command Update (DCU) is Dell's driver and BIOS management tool. While Windows Update handles OS patches, DCU handles everything specific to the Dell hardware: BIOS firmware, chipset drivers, display drivers, network adapter drivers, docking station firmware, and more. On HUIT machines, DCU is deployed and managed. You'll use it when a machine has hardware issues or you need to ensure drivers are current.

When to Run DCU
📶
Wi-Fi Issues
Intermittent drops, wrong speeds, can't see 6GHz networks. Wi-Fi adapter driver update often fixes these.
🖥️
Display Problems
Flickering, resolution issues, external monitor not working. Update Intel/AMD/Nvidia display drivers via DCU.
🔋
Battery/Power
Bad battery life, throttling, thermal issues. BIOS and EC firmware updates address power management.
⌚
New Machine Prep
Every new Dell should get a DCU pass before deployment. Ships with outdated drivers that need updating.
Running DCU
  1. Search "Dell Command Update" in Start Menu or find it in Start → All Apps
  2. Click Check for Updates — DCU scans the hardware and contacts Dell's servers
  3. Review the list. Updates are categorized as Critical, Recommended, and Optional
  4. Install at minimum all Critical and Recommended updates
  5. BIOS updates will require a restart — warn the user. BIOS updates are safe but the machine is briefly unusable during flash (usually 2–3 minutes, machine restarts itself)
  6. After restart, run DCU again to confirm no updates remain
⚠ BIOS Updates Require Power

If updating BIOS on a laptop, the machine must be plugged into AC power — not running on battery. DCU will refuse to install BIOS updates on battery. Make sure the charger is connected before starting.

💡 CLI Mode for Batch Runs

DCU has a command-line interface for scripting: dcu-cli.exe /applyUpdates -reboot=enable — useful when you need to update multiple machines or run it remotely via SCCM.

Group Policy & SCCM — Why Things Are Locked
Domain
Group Policy (GPO)

Group Policy is how Harvard IT centrally manages every domain-joined machine. When a setting is greyed out, an option is missing, or something "reset itself" after a reboot — that's usually Group Policy enforcing a standard. You cannot override GPO as a local admin without domain admin rights. Don't try to work around it — open a ticket if something needs changing.

🔒
Locked Settings
Greyed-out options in Settings or Control Panel = GPO enforced. Common: screen timeout, wallpaper, Windows Update settings, firewall rules.
🔄
Settings Reset After Reboot
User changes something, logs out, logs back in and it's reset. That's GPO pulling it back to the enforced state. Not a bug.
📦
Software Restrictions
GPO can block certain executables or require all software to come from approved paths. Users can't install random software without admin rights + approved sources.
🕐
Login Scripts
GPO runs scripts at login — mapping drives, configuring printers, pushing settings. If logins are slow, a GPO script may be timing out.
# Force GPO refresh immediately (run as admin)
gpupdate /force

# See what GPOs are applied to this machine
gpresult /r

# Full GPO report to HTML
gpresult /h C:\Temp\gpo-report.html
SCCM / Software Center

SCCM (System Center Configuration Manager) is how HUIT deploys and manages software across all machines. When a user needs software, the right path is through Software Center — not downloading from the internet.

  1. Open Software Center: Start Menu → search "Software Center" or look in All Apps
  2. Click Applications — shows all software available for this machine/user from HUIT
  3. Find the app → click Install. SCCM handles the download and installation silently
  4. Check Installation Status tab if install seems stuck — shows progress and any errors
  5. Updates tab shows pending OS and app updates that SCCM manages (separate from Windows Update)
💡 Software Not in Software Center?

If a user needs software that isn't in Software Center, they need to request it through a ServiceNow ticket. IT reviews, procures a license if needed, and deploys via SCCM. Don't install random software from the internet — it can conflict with security tools and violates Harvard policy.

Security Tools — Know What They Are, Don't Touch Them
🦅
CrowdStrike Falcon
Harvard's EDR (Endpoint Detection & Response). Monitors everything in real-time. If you see it in Task Manager, that's normal — don't stop it, disable it, or uninstall it. Ever.
📊
Nexthink
Harvard's experience monitoring agent. Tracks performance metrics, app crashes, and user experience data. Also normal — leave it alone. It's how HUIT proactively identifies issues.

Pull recent errors in one PowerShell command:
Get-EventLog -LogName System -Newest 50 -EntryType Error | Select TimeGenerated, Source, EventID, Message | Format-Table -AutoSize

Reading an Error — What to Look For
  1. Time — did the error happen at the same time as the user's reported issue?
  2. Source — which component failed? WSearch, Service Control Manager, Disk, NETLOGON each point to different problems
  3. Event ID — Google "Event ID [number] [Source]" for exact documentation. Microsoft Docs has every event ID documented with fix steps
  4. Message — read the full text. Often tells you exactly what failed and sometimes what to do about it
  5. Look for patterns — the same error repeating every few minutes = something is in a restart loop
