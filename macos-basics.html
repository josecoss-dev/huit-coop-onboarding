Finding the MAC Address on macOS
Daily Task
  1. Apple menu  → System Settings
  2. Click Network → select Wi-Fi or Ethernet → click Details...
  3. Hardware tab → MAC Address is shown here
  4. Copy and paste into ClearPass -- remember MAC Owner field = HUID (8-digit number), not HarvardKey
⚠ Private Wi-Fi Address (MAC Randomization)

Macs randomize their MAC per network by default. The randomized address won't work with ClearPass. To disable: System Settings → Network → Wi-Fi → Details → turn off Private Wi-Fi Address for the Harvard Secure network. Then use the real hardware MAC.

Terminal — zsh
% ifconfig en0 | grep ether
ether a4:c3:f0:1b:2e:7d
# en0 = Wi-Fi on most Macs. en1 is sometimes Ethernet.
% networksetup -listallnetworkservices
Wi-Fi
Ethernet
iPhone USB

Hold Option and click the Wi-Fi icon in the menu bar for a live connection panel.

Harvard Secure Wi-Fi on macOS
Common Setup
Certificate-Based Auth on Mac

Harvard Secure uses 802.1X certificate auth on Mac just like Windows. On macOS the certificate installs as a configuration profile via getonline.harvard.edu -- not into a certificate store.

  1. Disable Private Wi-Fi Address for Harvard Secure first: System Settings → Network → Wi-Fi → Details → toggle off Private Wi-Fi Address
  2. Open browser → go to getonline.harvard.edu → sign in with HarvardKey
  3. Site detects macOS → downloads a .mobileconfig profile file
  4. macOS shows "Profile Downloaded" → go to System Settings → Privacy & Security → Profiles → install it
  5. Enter Mac login password to authorize the install
  6. Connect to Harvard Secure in Wi-Fi settings → authenticates automatically using the cert
🌌
"Certificate not trusted" or keeps asking for credentials
Common
  1. Remove old profile: System Settings → Privacy & Security → Profiles → select Harvard profile → minus (-) to remove
  2. Open Keychain Access (Spotlight → Keychain Access) → search "harvard" → delete any old Harvard certs
  3. Re-run getonline.harvard.edu → install fresh profile
  4. Reconnect to Harvard Secure → should authenticate silently

Self Service — Software Deployment on Mac
Critical
Harvard's Mac App Deployment

On Windows you use Software Center (SCCM). On Mac you use Self Service, the Jamf-powered app portal. Jamf administrators assign apps and packages to organizational containers (Jamf groups), and Self Service surfaces only what's available for that specific machine's group. It's not a global catalog -- what you see depends on where the machine is enrolled.

📦
Finding Self Service
Applications folder → Self Service. Or Spotlight: Cmd+Space → "Self Service". It's usually pinned to the Dock on all Harvard-managed Macs.
👥
Container-Based Catalog
Jamf assigns apps to machine groups (containers). A machine in the Art Museum group sees different apps than one in Divinity. What's not listed isn't assigned -- submit a ServiceNow request.
🔒
Admin Credentials
Some installs prompt for admin credentials -- use the local admin account for that machine, same flow as Windows UAC with LAPS.
⚠️
No Internet Downloads
Same rule as Windows -- no random downloads. Software must come from Self Service or a ServiceNow request. CrowdStrike runs on Macs too.
🚫
Self Service shows error or won't check for updates
Common
  1. Verify the Mac is on the Harvard network or VPN -- Self Service needs to reach the Jamf server
  2. Force a Jamf check via Terminal: sudo /usr/local/munki/managedsoftwareupdate --checkonly
  3. Kick the Jamf launch daemon: sudo launchctl kickstart -k system/com.googlecode.munki.managedsoftwareupdate
  4. Restart Self Service after running the above

Terminal — Essential Commands for Mac Support
Critical
Opening Terminal

Cmd+Space → type "Terminal" → Enter. Or: Finder → Go → Utilities → Terminal. Shell is zsh on modern Macs (bash on older ones). The prompt looks like hostname % not C:\>.

Terminal — Network Commands
# Show Wi-Fi IP, subnet, router
% networksetup -getinfo Wi-Fi
IP address: 10.243.44.132
Subnet mask: 255.255.252.0
Router: 10.243.44.1
# Get Wi-Fi MAC address
% ifconfig en0 | grep ether
ether a4:c3:f0:1b:2e:7d
# Test connectivity (Ctrl+C to stop)
% ping -c 4 8.8.8.8
PING 8.8.8.8: 56 data bytes
64 bytes from 8.8.8.8: icmp_seq=0 ttl=118 time=12.4 ms
--- 8.8.8.8 ping statistics ---
4 packets transmitted, 4 received, 0% packet loss
# Trace route (not tracert like Windows)
% traceroute 8.8.8.8
# DNS lookup
% nslookup harvard.edu
Server: 10.243.44.1
Address: 10.243.44.1#53
Non-authoritative answer:
Name: harvard.edu
Address: 23.185.0.2
# Release and renew DHCP
% sudo ipconfig set en0 DHCP

Terminal — Admin Commands
# Elevate to root shell (use sparingly)
% sudo -s
Password: [enter admin password]
# Force Jamf check-in
% sudo jamf policy
Checking for policies triggered by "recurring check-in"...
Executing Policy Update Inventory...
# Clear quarantine flag on a downloaded app
% sudo xattr -cr /Applications/AppName.app
# List all users on the machine
% dscl . list /Users | grep -v ^_
admin
jose_coss
root
# Flush DNS cache
% sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
⚠ sudo = Admin on Mac

Commands prefixed with sudo run as root -- the Mac equivalent of running as administrator. Use with the local admin account credentials. Unlike Windows UAC, there's no graphical prompt -- the Terminal just asks for the password inline.

Jamf Pro — macOS MDM at Harvard
Reference
What Jamf Does

Jamf Pro is Harvard's MDM for Mac fleet management -- equivalent to SCCM on Windows. Jamf deploys software, enforces security policies, manages FileVault encryption, tracks inventory, and pushes configuration profiles. It runs silently in the background on every Harvard-managed Mac.

🔄
Jamf Check-In
Macs check in with Jamf at login and periodically. Like GPO refresh on Windows. Force a check-in: sudo jamf policy in Terminal.
🔒
FileVault
Jamf enforces full disk encryption via FileVault on all Harvard Macs. NEVER disable it. If FileVault is off, flag it immediately -- security violation.
📋
MDM Profiles
Jamf pushes configuration profiles (Wi-Fi certs, security settings, etc.). View installed profiles: System Settings → General → Device Management.
🔗
Re-Enrollment
If a Mac loses MDM management: go to the Harvard Jamf enrollment URL (get from your supervisor), sign in with HarvardKey, follow the enrollment steps.
⚠ Co-ops Don't Have Jamf Console Access

The Jamf admin console is a senior IT tool. Co-ops use sudo jamf policy from Terminal to force a check-in, and check System Settings for profile status -- that's the extent of Jamf interaction at your level. Escalate anything beyond that.

✓ How to Confirm a Mac Is Managed

System Settings → General → Device Management. You should see a management profile listed. If the Device Management section shows nothing, the Mac may not be enrolled -- escalate to a senior tech.

Common macOS Issues — Fix Playbook
Playbook
🐢
Slow Mac — high CPU or spinning beachball
High
  1. Open Activity Monitor (Spotlight: Cmd+Space → Activity Monitor) → CPU tab → sort by % CPU descending
  2. Find the culprit process. kernel_task at high CPU = thermal throttling -- make sure the Mac is on a hard surface, vents not blocked, not on fabric
  3. If it's a user app: double-click it → click Quit or Force Quit
  4. Check Memory tab: Swap Used > 1GB means RAM is full -- close apps or the machine may need a RAM upgrade request
  5. Run Software Update: Apple menu → System Settings → General → Software Update -- pending updates can cause background CPU spikes
🌌
Wi-Fi won't connect or drops constantly
High
  1. Turn Wi-Fi off then back on: click Wi-Fi icon → Turn Wi-Fi Off → wait 5 seconds → Turn Wi-Fi On
  2. Forget and rejoin Harvard Secure: System Settings → Network → Wi-Fi → Known Networks → find Harvard Secure → Forget → reconnect
  3. If cert error: remove Harvard Wi-Fi profile (Privacy & Security → Profiles) and re-run getonline.harvard.edu
  4. Check Private Wi-Fi Address is off for Harvard Secure -- randomized MAC = ClearPass rejection
  5. Run Option+Click Wi-Fi icon: check RSSI. Below -75 dBm = weak signal, physical location issue
🔒
Keychain keeps prompting for passwords
Common
  1. Open Keychain Access (Spotlight → Keychain Access)
  2. Search for the app or service that's prompting (e.g., "harvard", "outlook")
  3. Find duplicate or expired entries → delete them
  4. Re-enter credentials when prompted and check "Always Allow" or "Save in Keychain"
  5. If the login keychain itself is locked: Keychain Access → right-click login keychain → Unlock Keychain → enter Mac login password
🚫
"App is damaged and can't be opened" or unverified developer
Common
  1. First: right-click the app → Open (not double-click) -- macOS lets you override Gatekeeper once this way
  2. If still blocked: System Settings → Privacy & Security → scroll down → you'll see "[App] was blocked" → click Open Anyway
  3. For quarantine flag: Terminal → sudo xattr -cr /Applications/AppName.app
  4. If it's a Harvard-approved app that should be in Self Service: check Self Service first, avoid manual workarounds
🖨
Printer not working on Mac
Common
  1. System Settings → Printers & Scanners → select the stuck printer → click minus (-) to remove it
  2. Re-add the printer: click plus (+) → find the Crimson Print queue for the department (e.g., HDS-Crimson-Print for Divinity) on the network
  3. If print job is stuck: click the printer → Open Print Queue → delete the stuck job
  4. Crimson Print PIN release: same Harvard PIN used on Windows -- reset at pin.harvard.edu if forgotten
👤
macOS profile corruption — weird behavior for one user
Less Common
  1. Create a new test user: System Settings → Users & Groups → Add Account (requires admin)
  2. Log into the new test account -- if the issue disappears, the problem is profile-specific to the original user
  3. For a corrupted user profile: back up the user's home folder, delete the account, recreate it, restore data
  4. Escalate to a senior tech before deleting any accounts -- data loss risk
💾
macOS update stuck or failing
Occasional
  1. Check disk space: Apple menu → About This Mac → Storage -- need at least 20GB free for a major update
  2. Restart and try again: Apple menu → Restart → run Software Update after fresh boot
  3. "This Mac can't be updated": check the minimum macOS requirement for the update -- older hardware may not support it. File a hardware refresh request.
  4. If update downloads but won't install: delete the update package from /Library/Updates and re-download

Crimson Print on macOS — Department Queues
Common Setup
How Printing Works on Mac at Harvard

Crimson Print (PaperCut NG) works the same on Mac as Windows -- jobs are logged, charged to department codes, and released at the printer with a Harvard PIN. Each department has its own named queue. The Mac setup just requires adding the queue manually.

📋
Department Queues
Each department has a named Crimson Print queue. Divinity School: HDS-Crimson-Print. Art Museum: check with your supervisor for the queue name used at HUAM.
📱
PrintAnywhere on Mac
Upload to print.harvard.edu in any browser. Download and install the PrintAnywhere client if a printer-specific app is needed. Release at any Harvard PrintAnywhere printer with your PIN.
print.harvard.edu
🔒
Harvard PIN
Same 4-digit PIN as Windows. Reset at pin.harvard.edu. PIN is not the HarvardKey password. Users forget this constantly.
pin.harvard.edu
Adding a Crimson Print Queue on Mac
  1. System Settings → Printers & Scanners → click + to add a printer
  2. In the Add Printer dialog → click the IP tab at the top
  3. Address: enter the Crimson Print server hostname (get from your supervisor or department IT liaison)
  4. Protocol: Internet Printing Protocol - IPP
  5. Queue: enter the department queue name (e.g., HDS-Crimson-Print)
  6. Name it something recognizable → click Add → test with a small print job
💡 Jamf Deploys Printers Too

On fully managed Macs, Jamf may push the correct Crimson Print queue automatically at enrollment. Check Printers & Scanners before manually adding -- it might already be there from a Jamf policy.

Supporting Macs vs PCs at Harvard — What's Different
Context
User Expectations Are Different

Mac users at Harvard -- especially at the Art Museum -- often know their machines well. They're creative professionals, researchers, or faculty who have used Macs for years. Your approach needs to match that.

🎨
Art Museum
Mostly Macs. Mix of MacBook Pros and Mac Studios. Users are curators, researchers, admin -- non-technical but confident with their specific Mac workflows. Don't talk Windows at them.
📚
Divinity School
Mixed fleet. Faculty may have older Macs. Students use personal devices too. Check whether the machine is Harvard-managed before trying Jamf or Self Service.
📚
Lamont Library
Mostly Windows at the circulation desk. Macs in some staff offices. Public computers are a mix -- don't install anything on public machines without supervisor approval.
💬
Language Matters
"Control Panel" doesn't exist. Say "System Settings". "Programs" are "Applications" or "apps". "C drive" is "Macintosh HD". Adjust your vocabulary or you'll lose them.
💡 Network File Shares on Mac

Macs connect to Harvard network shares via SMB, same as Windows. Finder → Go → Connect to Server → type smb://server.harvard.edu/sharename → enter HarvardKey credentials. The share mounts like a drive in Finder. Users can also drag it to the Dock for quick access.

Apple School Manager — iPhones, iPads & Shared Devices
Reference
What Apple School Manager Is

Apple School Manager (ASM) is Apple's cloud-based management portal for educational institutions. Harvard uses it to manage the Apple ecosystem at scale -- not just Macs, but also iPhones and iPads used in departments. ASM integrates with Jamf Pro, which does the actual device configuration and app deployment.

🏢
What ASM Does
Links Harvard's Apple device purchases to Harvard's Jamf instance automatically. When a new iPhone or iPad is unboxed, it enrolls into Jamf on first power-on -- no manual enrollment needed if purchased through Apple or an authorized reseller.
📱
iPhones & iPads
Departments sometimes have shared or role-based iPhones and iPads (e.g., Art Museum gallery guides, Divinity research devices). These are managed through Jamf via ASM -- same container/group logic as Macs.
📦
App Deployment on iOS
Apps on managed iPhones/iPads are pushed silently through Jamf -- users don't download from the App Store manually. The apps just appear. Removal is also controlled by Jamf admins.
👤
Shared Device Mode
Some iPads run in Shared Device Mode -- multiple users log in and out with their own credentials, but the device config stays consistent. Common in library or lab settings.
🔒
Activation Lock
ASM bypasses Activation Lock for Harvard-owned devices. If a device is stuck at Activation Lock after a wipe, it means it wasn't enrolled through ASM -- escalate to a senior tech, this requires Apple ID resolution.
⚠️
Personal vs Harvard Device
If a user brings in a personal iPhone for IT help, do NOT enroll it in Jamf or ASM. Harvard MDM should only be on Harvard-owned hardware. Personal devices = advise, don't manage.
💡 Co-op Role with iOS Devices

Co-ops don't have Jamf or ASM admin access for iOS -- that's senior IT. Your role: help users connect to Harvard Secure Wi-Fi, set up Harvard email in the Mail app, troubleshoot Duo MFA on the device, and escalate anything requiring MDM changes. If an iPad is acting up and it's Harvard-owned, confirm it's managed (Settings → General → VPN & Device Management) then escalate.

⚠ Activation Lock on a Wiped Device

If someone wipes a Harvard iPhone or iPad without going through Jamf first, it may get stuck at Activation Lock with a personal Apple ID. This is not fixable at the co-op level -- escalate immediately. Going through Jamf to wipe a device avoids this entirely.
