⚠ The #1 Sign-In Mistake

Users frequently sign into M365 apps with a personal @outlook.com, @hotmail.com, or @gmail.com account by accident. They'll see their old personal OneDrive, not Harvard resources. Always verify they're signed in as firstname_lastname@harvard.edu — check the account avatar in the top-right of any M365 app.

💡 HarvardKey vs Harvard Email

Users sometimes confuse their HarvardKey username (used for VPN, ClearPass, Harvard systems) with their email address. HarvardKey is just the username portion — firstname_lastname. The full email adds @harvard.edu. Both route to the same identity. When signing into M365: use the full email address format.

👥

A group of people — like a department or project group. Has channels inside it. Must be a member to see it. Created by IT or team owners.
#

A topic thread inside a Team. General channel is always there. Others are created by team owners. Channels can be public (to the team) or private (subset of members).

Direct messages between people. Not part of any Team. Good for quick 1:1 or small group conversations. Doesn't require team membership.

Common Teams Issues
❓
"I can't find my team / I don't see my channels"
Common
  1. Check the account in the top-right avatar — are they signed in as @harvard.edu or a personal account? If personal, sign out and sign in with Harvard credentials.
  2. Check if the team is hidden: click "..." next to Teams in the left sidebar → click hidden teams → show the team they need.
  3. If a channel is missing from a team they're in: the channel is probably private. Team owner must add them to the private channel separately.
  4. If they can't find the team at all: they need to be added by the team owner. IT cannot add users to Teams — only team owners can.
💥
Teams won't load or keeps crashing
High
  1. Fully quit Teams: right-click the Teams icon in system tray → Quit. Don't just close the window.
  2. Clear Teams cache: navigate to %AppData%\Microsoft\Teams → delete the contents of the Cache, blob_storage, databases, and GPUCache folders.
  3. Restart Teams. It will rebuild the cache on launch (takes ~30 seconds).
  4. If still broken: sign out of Teams completely → sign back in with Harvard credentials.
  5. Last resort: uninstall Teams via Settings → Apps, reinstall from portal.office.com.
⚡ Quick Fix Command

taskkill /F /IM Teams.exe then navigate to %AppData%\Microsoft\Teams and clear the Cache folder. Faster than hunting through system tray.

🎤
Audio/video not working in meetings
Common
  1. In Teams: click your avatar → Settings → Devices. Verify the correct microphone and speaker are selected — especially after a new headset or dock is connected.
  2. Check Windows Sound settings: right-click speaker icon in taskbar → Open Sound settings → verify input and output devices match what Teams shows.
  3. If using a docking station: Dell docks sometimes require DCU driver updates for audio to work correctly.
  4. Test with the built-in mic/speakers first to rule out the device vs Teams.

Outlook & Harvard Email
Critical
First-Time Outlook Setup

Outlook on a domain-joined Harvard machine will usually auto-configure via Autodiscover — it detects the Harvard Exchange/M365 settings automatically when the user enters their Harvard email. No manual server settings needed.

  1. Open Outlook. Enter full Harvard email: firstname_lastname@harvard.edu
  2. Autodiscover handles the rest — Outlook contacts Microsoft's servers and pulls configuration.
  3. Duo MFA prompt will appear — approve it.
  4. If Autodiscover fails (rare on domain machines): File → Account Settings → Add Account → manual setup — server type: Microsoft 365, email as above.
Shared Mailboxes

Departments often use shared mailboxes — a single email address monitored by multiple people (e.g., artmuseum-it@harvard.edu). Users need to add them manually.

  1. File → Account Settings → Account Settings → double-click the Harvard account
  2. More Settings → Advanced tab → Add
  3. Type the shared mailbox email address → OK → Apply
  4. The shared mailbox appears in the left folder pane below their own mailbox
⚠ Send As Permissions

"I can see the shared mailbox but can't send emails from it" — the user needs Send As permission on the mailbox, not just access. This is granted by the mailbox owner or IT admin via M365 admin portal — co-ops cannot grant this themselves.

Common Outlook Issues
🔞
"I'm not receiving emails" or emails going missing
High
  1. Check Focused Inbox: Outlook splits inbox into Focused and Other tabs. Emails may be hiding in Other. View → Show Focused Inbox (toggle off to see everything together).
  2. Check Junk Email folder — Proofpoint (Harvard's email filter) may have quarantined legitimate email.
  3. Check Rules: File → Manage Rules & Alerts. User may have accidentally created a rule moving emails.
  4. Check Online Archive: older emails may have been auto-archived. Look for the "Online Archive" folder in the left pane.
  5. Send a test email from another account to confirm delivery is working.
🔒
Outlook keeps prompting for password
Common
  1. Open Credential Manager: Start → search "Credential Manager" → Windows Credentials tab.
  2. Find and remove any entries containing "MicrosoftOffice", "outlook", or "microsoft.com".
  3. Restart Outlook — it will prompt for credentials once cleanly and store them properly.
  4. If still prompting: HarvardKey password may have recently changed and the cached token is stale. Have the user re-authenticate through portal.office.com first.
🔌
Outlook profile corruption — won't open or errors on launch
High
  1. Try safe mode first: Win+R → outlook.exe /safe — if it opens, a COM add-in is the culprit. Disable all add-ins in File → Options → Add-ins.
  2. Reset nav pane: outlook.exe /resetnavpane — fixes "cannot open the Outlook window" errors.
  3. Rebuild the Outlook profile: Control Panel → Mail → Show Profiles → Add a new profile → set it as default → launch Outlook fresh with the new profile.
  4. If the .ost file is corrupted: close Outlook, navigate to %LocalAppData%\Microsoft\Outlook, rename or delete the .ost file. Outlook rebuilds it by syncing from server.

OneDrive for Business — Harvard Cloud Storage
Common Issue
Harvard OneDrive vs Personal OneDrive

Harvard users get 1TB OneDrive for Business tied to their Harvard account. This is completely separate from any personal OneDrive they might have with a @hotmail.com account. If a user is signed into the wrong account, they'll see the wrong files. Always verify the OneDrive account is @harvard.edu.

☁️
Files On-Demand
Harvard enables Files On-Demand by default. Files show a cloud icon — they exist in OneDrive but aren't downloaded locally. Click to open, it downloads instantly. Not missing.
🔄
Sync Status Icons
Blue cloud = online only. Green check = synced locally. Sync arrows = currently syncing. Red X = sync error — needs attention. Check the system tray OneDrive icon.
🖥️
Known Folder Move
Harvard policy moves Desktop, Documents, and Pictures to sync with OneDrive. This means user files survive a machine wipe — but also means Desktop is in the cloud.
⚠️
When Users Leave
Harvard deactivates OneDrive after a grace period when employment ends. Managers should transfer ownership of shared files before departure. Warn users to back up personal items.
Common OneDrive Issues
👁️
"My files disappeared" — OneDrive Files On-Demand
Very Common
  1. Navigate to the folder in File Explorer. If you see cloud icons next to files, they're online-only — they exist, just not downloaded.
  2. Right-click a file → "Always keep on this device" to force a local download.
  3. To download an entire folder: right-click the folder → "Always keep on this device".
  4. Verify OneDrive is signed in to the Harvard account: right-click the OneDrive icon in system tray → Settings → Account tab.
🔀
OneDrive stuck syncing or showing red X
Common
  1. Right-click OneDrive tray icon → Pause Syncing (2 hours) → then Resume Syncing. Often clears a stuck queue.
  2. If red X error: click the OneDrive icon → read the error message — it usually tells you exactly which file is causing the problem (file name too long, illegal character, file in use).
  3. File name issues: Windows has a 260-character path limit. Files with names like CON, PRN, AUX, or trailing spaces won't sync.
  4. If completely broken: right-click OneDrive tray → Settings → Account → "Unlink this PC" → sign back in. OneDrive re-syncs from scratch.
🗂️
Two OneDrive icons in File Explorer (personal + Harvard)
Cosmetic

One says "OneDrive — Harvard University", the other says "OneDrive — Personal". The personal one is their own @hotmail.com or @gmail.com account. They should only use the Harvard one for work files. To remove the personal: right-click OneDrive tray icon → Settings → Account → Unlink personal account. This removes the personal sync, not the personal OneDrive itself.

SharePoint — Department File Sharing
Reference
What SharePoint Is at Harvard

SharePoint is Harvard's platform for team and department file sharing. When a user clicks the Files tab in a Teams channel, they're looking at SharePoint. When a department has a shared drive accessible via a URL, that's also SharePoint. It's the backbone of file collaboration.

👥
Site Owner
Full control. Can add/remove members, change permissions, create subsites. Usually a department head or designated IT liaison.
✎️
Site Member
Can read and edit files in the site. Default role for staff added to a department SharePoint. Cannot change permissions.
👁️
Site Visitor
Read-only access. Can view but not edit files. Used for reference documents or public-facing department sites.
⚠ IT Cannot Grant SharePoint Access

"I need access to this SharePoint site" — co-ops and HUIT cannot grant access to SharePoint sites. Only the site owner can add members. Direct the user to contact whoever runs the site (usually their manager or department admin). IT can only intervene at the tenant-admin level for serious issues.

Sync SharePoint to Local OneDrive
  1. Navigate to the SharePoint site in a browser (share the URL with the user)
  2. Click Sync in the top toolbar — this opens OneDrive and sets up the sync
  3. The SharePoint library now appears in File Explorer under OneDrive — Harvard University
  4. Files work like local files but sync with the SharePoint site
💡 Teams Files Tab IS SharePoint

Every Teams channel has a Files tab. Behind the scenes, that's a SharePoint document library. If a user needs to access those files in File Explorer, they can click the Files tab → "Open in SharePoint" → then click Sync. The files appear locally in OneDrive just like personal files.

Critical
How Duo Works at Harvard

Every Harvard M365 sign-in requires a Duo second factor after the HarvardKey password. Duo is non-negotiable — there is no MFA-exempt path for regular users. Co-ops will handle Duo questions constantly.

Scenarios You'll Handle
📱
New Device Enrollment
User got a new phone. Send them to go.harvard.edu/duo — they can add new devices through the Duo self-service portal. Old device stays active until removed.
go.harvard.edu/duo
🚫
Lost/Broken Phone
User can't get a Duo push. They need a temporary bypass code from the HUIT helpdesk. Call 617-495-7777 or submit an emergency ticket. Cannot be self-served.
617-495-7777
🔌
Duo Passcode
If no internet on the phone, Duo app generates a 6-digit TOTP passcode. Open Duo app → tap the key icon → use that passcode at the login page.
⚠️
Duo Fatigue Attack
User gets repeated Duo pushes they didn't initiate. This means someone has their password and is trying to log in. Deny every push. Change password immediately. Escalate to security@huit.harvard.edu.
🚨 Duo Fatigue is a Real Attack

Attackers who have stolen a Harvard password will send dozens of Duo push requests hoping the user gets tired and taps Approve by mistake. If a user reports unsolicited Duo pushes: (1) tell them to deny everything, (2) have them change their HarvardKey password immediately at key.harvard.edu, (3) report to security@huit.harvard.edu. Do not wait.

Common M365 Fixes — Your Daily Playbook
Critical

These are the calls you'll get every week. Expand each one for the exact fix procedure.

💥
Teams cache cleared but still won't launch
High
  1. Kill all Teams processes: taskkill /F /IM Teams.exe
  2. Delete entire Teams folder: %AppData%\Microsoft\Teams — delete all contents
  3. Also check: %LocalAppData%\Microsoft\Teams — clear this too
  4. If on new Teams (Teams 2.0): look in %LocalAppData%\Packages\MSTeams_8wekyb3d8bbwe
  5. Relaunch Teams. If still failing: uninstall via Apps & Features, redownload from teams.microsoft.com
🔒
Outlook not connecting — spinning or offline
High
  1. Check network: can they reach outlook.office365.com in a browser? If not, it's a network issue, not Outlook.
  2. Clear cached Office credentials: Credential Manager → Windows Credentials → remove all "MicrosoftOffice" entries.
  3. Sign out of Office: any Office app → File → Account → Sign Out → sign back in.
  4. Delete and recreate the Outlook profile via Control Panel → Mail → Show Profiles.
  5. Check status.harvard.edu — if there's an M365 incident, there's nothing to fix locally.
🔀
OneDrive won't sync — reset it
Common
  1. Right-click OneDrive tray icon → Settings → Account → Unlink this PC.
  2. Close OneDrive completely.
  3. Reopen OneDrive from Start menu → sign back in with Harvard email.
  4. OneDrive re-syncs from cloud. No data is lost — it's already in the cloud.
📩
Shared mailbox not appearing in Outlook
Common
  1. Confirm the user has been granted access to the shared mailbox (by the mailbox owner, not IT).
  2. File → Account Settings → Account Settings → double-click the Harvard Exchange account.
  3. More Settings → Advanced → Add → type the shared mailbox address → OK.
  4. If it doesn't appear after restarting Outlook: remove and re-add the user's permission on the shared mailbox (requires mailbox owner or Exchange admin).
🚫
"Your organization's policies are preventing..." error
Policy

This is a Conditional Access or Microsoft Intune policy blocking a feature or app. Common triggers: trying to access M365 from a personal (unmanaged) device, trying to use a feature restricted by HUIT policy (e.g., forwarding all email externally). Co-ops cannot override these policies — they're tenant-level security controls. If the restriction is blocking legitimate work, the user needs to submit a request through ServiceNow for IT policy review.

M365 Admin Portal — What Exists, What Co-ops Can Do
Reference
admin.microsoft.com

The M365 Admin Portal is where Harvard IT manages users, licenses, groups, and settings. Most co-ops won't have admin portal access — but knowing what's there means you know exactly what information to pull and send to Tier 2.

👥
User Management
View user accounts, check license assignment, see sign-in status (enabled/blocked). Password reset for M365 goes through HarvardKey self-service, NOT the admin portal.
admin.microsoft.com/Users
🏢
Groups & Distribution Lists
M365 groups, distribution lists, mail-enabled security groups. Adding users to distribution lists is often an admin task co-ops can handle if given limited admin role.
📋
License Assignment
Which licenses a user has (E3, E5, Power BI, etc.). License changes go through HUIT process — submit a ticket, don't self-serve.
📈
Service Health
admin.microsoft.com/ServiceHealth — live status of all M365 services. Check this before assuming it's a local issue.
⚠ Password Resets Are NOT in M365 Admin

Harvard password resets happen through HarvardKey at key.harvard.edu — not through the M365 admin portal. This confuses a lot of people. The HarvardKey system controls authentication; M365 just trusts it. Co-ops cannot reset HarvardKey passwords — users must use the self-service tool or contact the Identity team.
