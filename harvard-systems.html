HarvardKey -- Your Users' Master Key
Foundation
What HarvardKey Is

HarvardKey is Harvard's Single Sign-On identity system. One set of credentials authenticates into virtually every Harvard digital system: M365 email, VPN, Canvas, ClearPass, library resources, and more. When a user can't log into anything at Harvard, HarvardKey is almost always where to start.

HarvardKey Disabled
Disabled = employment ended or security hold. Different from locked. Requires HUIT Identity team or HR action. Co-ops cannot re-enable.
↻
Password Sync
HarvardKey password syncs to Windows domain and M365. If password changes, cached Windows credentials need updating too -- lock/unlock screen to trigger sync.
⚠ Co-ops Cannot Reset HarvardKey Passwords

Password reset is self-service at key.harvard.edu, or through the HUIT Identity team helpdesk. Co-ops do not have access to reset HarvardKey accounts. Submit a ticket to Identity if the user is locked out and can't self-serve.

💡 HarvardKey vs HUID -- Critical Difference

HarvardKey = username string (jose_coss) used for system login. HUID = 8-digit number on the Harvard ID card (12345678) used for ClearPass, library, HR. Completely different. Users confuse these constantly -- clarify before troubleshooting.

GlobalProtect VPN -- Accessing Harvard Remotely
Critical
Why VPN Exists

Many Harvard systems are only accessible from the Harvard network -- either on campus or via VPN. VPN creates an encrypted tunnel from a remote device into Harvard's network. Client: Cisco AnyConnect. Gateway: vpn.harvard.edu.

💡 Split Tunnel

Harvard VPN is split-tunnel -- only Harvard-bound traffic goes through the VPN. Regular internet (Google, Netflix, etc.) goes directly through your ISP. VPN doesn't slow down general browsing.

Common VPN Issues
🚫
VPN stuck on "Connecting..." or won't connect
High
  1. Check internet first: can they reach google.com? No internet = VPN can't connect regardless.
  2. Disconnect and reconnect in GlobalProtect -- wait 5 seconds between.
  3. Restart AnyConnect service: Task Manager → Services tab → find vpnagent or acumbrellaagent → right-click → Restart.
  4. Elevated PowerShell: net stop vpnagent then net start vpnagent.
  5. Uninstall and reinstall AnyConnect from vpn.harvard.edu (requires HarvardKey to authenticate and download).
🌐
VPN connects but can't reach a specific system
Common
  1. Confirm VPN is actually connected -- AnyConnect icon shows connected shield.
  2. Verify the target system actually requires VPN -- many Harvard portals are cloud-based and don't need it.
  3. Try pinging a Harvard internal address to confirm tunnel routing.
  4. Escalate to HUIT networking if VPN confirms connected but specific resource is unreachable -- may need firewall rule or different VPN gateway profile.

Harvard Printing -- PaperCut & PrintAnywhere
Common Issue
How Harvard Printing Works

Harvard uses Crimson Print (powered by PaperCut NG) for print management. Every job is logged, charged to a department code, and released at the printer with a Harvard PIN. Each department has its own named print queue — for example, Divinity School is HDS-Crimson-Print, Central Administration is CADM-Crimson-Print. PrintAnywhere at print.harvard.edu lets users upload documents and release them at any Harvard printer.

📋
Crimson Print
Harvard's print management system (PaperCut NG backend). Each department has its own named queue. Jobs are tracked, charged per page, and expire after 24 hours uncollected.
📱
PrintAnywhere
Upload any document at print.harvard.edu, release at any Harvard PrintAnywhere printer with your PIN. No driver install needed.
print.harvard.edu
🔒
Harvard PIN
4-digit numeric PIN to release print jobs. Separate from HarvardKey password. Reset at pin.harvard.edu. Self-service only -- co-ops cannot reset PINs.
pin.harvard.edu
💳
Department Codes
Print costs charged to department budget codes. User doesn't pay directly -- their department does. Color costs more than B&W.
Common Print Issues
📋
"My print job disappeared"
Quick

PaperCut expires uncollected jobs after 24 hours. The job is gone -- reprint it. Tell users to collect same day they print.

🔒
Wrong PIN / Forgotten PIN at printer
Quick

Harvard PIN is NOT the HarvardKey password. Separate 4-digit PIN managed at pin.harvard.edu. Users reset it themselves -- co-ops cannot reset PINs on their behalf.

🖨
Job stuck in Windows print queue
Common
  1. Don't try to delete from the Windows queue -- PaperCut manages the job.
  2. Cancel from PaperCut web interface or at the printer touchscreen.
  3. If Windows queue is also stuck: Stop Spooler service, clear spool folder, restart Spooler (see Windows Basics, Mod 07, Fix 6).

Harvard Status Page -- Outage Awareness
Critical
status.harvard.edu

Check this before you touch anything when users report broad issues. If it's on the status page, you don't troubleshoot -- you acknowledge, document, and wait.

●
Operational
Service working normally. User issues are likely local to their machine or account.
●
Degraded
Partial outage or performance issues. Some users affected. Don't escalate locally -- central team is on it.
●
Major Outage
Service down. Do not troubleshoot. Acknowledge, point to status page, log a ticket referencing the incident number.
✓ The Right Workflow

User reports issue → Check status.harvard.edu → If outage: "There's a known issue with [service] that Harvard IT is actively working on" → Log ticket, reference the incident → No further local troubleshooting.

Common Harvard System Issues -- Fix Playbook
Playbook
🚫
"I can't log into any Harvard system"
High
  1. Check status.harvard.edu for a HarvardKey or identity system outage.
  2. Have them try key.harvard.edu directly. If it loads and they can log in, account is active -- issue is application-specific.
  3. If key.harvard.edu also fails: account may be locked. Try self-service at key.harvard.edu/unlock. If that fails, escalate to HUIT Identity team.
  4. Did they recently change their password? New password may not have synced to Windows -- lock and unlock screen to force sync.
  5. Do they have a Duo device enrolled? No Duo = no MFA = can't log in anywhere.
📱
"I forgot my PIN and can't print"
Quick

Harvard PIN reset is self-service only at pin.harvard.edu. Co-ops cannot reset PINs. Requires HarvardKey login. Takes under 2 minutes.

🌐
VPN connects but a specific system is still unreachable
Common
  1. Confirm VPN is actually connected -- check AnyConnect icon state.
  2. Verify the target system actually requires VPN -- many Harvard portals are cloud-based.
  3. Try pinging the system hostname to confirm routing.
  4. Escalate to HUIT networking if VPN confirmed connected but specific resource unreachable -- may need firewall rule or different VPN gateway.
🔒
"My Harvard ID doesn't open the door"
Refer

Physical card access is managed by HUPD and the Access Control office -- not HUIT IT. Refer to department admin or HUPD. Lost/damaged cards: Holyoke Center HUID Office. This is not an IT issue.

📁
"I can't access the department SharePoint"
Common

SharePoint access is granted by the site owner -- not IT. Co-ops cannot grant SharePoint permissions. Direct the user to contact their manager or the department IT liaison who owns the site.
